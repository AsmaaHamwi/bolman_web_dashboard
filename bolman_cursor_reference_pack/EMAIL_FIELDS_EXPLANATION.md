# شرح حقول البريد الإلكتروني عند إنشاء شركة

## المشكلة
عند إنشاء شركة في داشبورد النظام، يطلب البريد الإلكتروني مرتين:
1. **البريد الإلكتروني للشركة** (Company Email)
2. **البريد الإلكتروني لصاحب الشركة** (Owner Email)

---

## الفرق بين البريدين

### 1. البريد الإلكتروني للشركة (Company Email)
- **الحقل**: `company.email`
- **جدول قاعدة البيانات**: `companies.email`
- **الاستخدام**: بريد تواصل عام للشركة (مثل بريد الدعم أو البريد الإداري)
- **الخصائص**:
  - اختياري (يمكن تركه فارغاً)
  - لا يستخدم للمصادقة (Authentication)
  - يظهر في قائمة الشركات للعرض فقط

### 2. بريد صاحب الشركة (Owner Email) ⭐
- **الحقل**: `owner.email`
- **جدول قاعدة البيانات**: 
  - `auth.users.email` (في نظام المصادقة)
  - `public.users.email` (في جدول الملف الشخصي)
- **الاستخدام**: **بريد تسجيل الدخول الرئيسي**
- **الخصائص**:
  - إجباري (يجب ملء هذا الحقل)
  - يستخدم للمصادقة والدخول للنظام
  - هو البريد الذي سيستخدمه صاحب الشركة لتسجيل الدخول

---

## مثال توضيحي

عند إنشاء شركة "أسطول الميدان":

| الحقل | القيمة | الاستخدام |
|------|--------|----------|
| **Company Email** | support@astalmidon.com | بريد للعملاء للتواصل |
| **Owner Email** | salem@astalmidon.com | **بريد الدخول** |
| **Owner Name** | سالم محمد | - |
| **Owner Password** | Pa@ssw0rd123 | كلمة المرور |

### عند تسجيل الدخول:
صاحب الشركة يستخدم:
- **البريد**: `salem@astalmidon.com` ✅
- **كلمة المرور**: `Pa@ssw0rd123` ✅

## الهيكل التقني

```
┌─────────────────────────────────────────┐
│        جدول الشركات (companies)        │
├─────────────────────────────────────────┤
│ id              │ UUID                  │
│ name            │ "أسطول الميدان"      │
│ email           │ support@...com        │ ← Company Email
│ phone           │ "0541234567"          │
│ owner_user_id   │ UUID (مرجع للمالك)   │
└─────────────────────────────────────────┘
         ↓ يشير إلى
┌─────────────────────────────────────────┐
│        جدول المستخدمين (users)         │
├─────────────────────────────────────────┤
│ id              │ UUID                  │
│ full_name       │ "سالم محمد"           │
│ email           │ salem@...com          │ ← Owner Email
│ phone           │ "0501234567"          │
│ role            │ "company_owner"       │
│ status          │ "active"              │
└─────────────────────────────────────────┘
         ↓ يرتبط مع
┌─────────────────────────────────────────┐
│      جدول المصادقة (auth.users)        │
├─────────────────────────────────────────┤
│ id              │ UUID (نفس id المستخدم)
│ email           │ salem@...com          │ ← يستخدم للدخول
│ password_hash   │ ••••••••••••••        │
└─────────────────────────────────────────┘
```

## ملخص الإجابة

| السؤال | الإجابة |
|------|--------|
| **لماذا يطلب الايميل مرتين؟** | الأول للشركة والثاني لصاحبها (المالك) |
| **الفرق بينهما؟** | الأول بريد الشركة العام، الثاني بريد الدخول الشخصي |
| **أيهم يستخدم صاحب الشركة للدخول؟** | **بريد صاحب الشركة (Owner Email)** فقط |
| **هل يمكن تركهما فارغين؟** | الأول اختياري، الثاني **إجباري** |

---

## الملاحظات المهمة ⚠️

1. **صاحب الشركة لا يمكنه الدخول ببريد الشركة**
   - إذا أدخلت `support@astalmidon.com` كبريد شركة
   - و `salem@astalmidon.com` كبريد المالك
   - فيجب استخدام `salem@astalmidon.com` للدخول

2. **كلمة المرور المؤقتة** 
   - الكلمة الافتراضية: `12345678`
   - يجب على صاحب الشركة تغييرها عند أول دخول

3. **الأمان**
   - بريد المالك محفوظ في نظام المصادقة Supabase
   - كلمة المرور لا تُخزن مباشرة، بل يُخزن الـ hash فقط
   - العملية آمنة عبر الدالة السحابية `create-company-with-owner`
